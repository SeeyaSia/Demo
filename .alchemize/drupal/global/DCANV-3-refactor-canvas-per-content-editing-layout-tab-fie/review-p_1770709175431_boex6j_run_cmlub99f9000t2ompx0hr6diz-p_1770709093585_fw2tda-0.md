### Change 2: Specify API communication path for field-scoped editability

**Requested:** Define query parameter name, controller code to read it, and note the auto-save path.

**Resolution:** Step 2.4 (lines 488-500) now specifies:
- **Query parameter:** `canvas_target_field` (string, optional)
- **Frontend flow:** React reads `drupalSettings.canvas.templateContext.targetFieldName` → appends `?canvas_target_field={field_name}` on API requests
- **Controller code:** `$this->requestStack->getCurrentRequest()->query->get('canvas_target_field')` in `ApiLayoutController::get()` at ~line 104
- **Propagation:** Passed as new optional parameter to `annotateEditableRecursive()`
- **Slot resolution:** Uses `getExposedSlots()[$target_field]` to find the target slot's `component_uuid`/`slot_name`, marks only those descendants as `editable: true`
- **Auto-save path:** Explicitly states ~line 519 needs the same treatment, and React must include the parameter on both GET and auto-save requests
- **Backward compatibility:** When absent, all slot content remains editable (preserving current behavior)

**Verdict:** Fully resolved. The specification is concrete and implementable.
