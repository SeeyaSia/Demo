## Phase 4: Quality Assessment

### Drupal Best Practices

- **Dependency injection**: All new services use constructor injection via autowiring (services.yml `_defaults: autoconfigure: true, autowire: true`). ✓
- **Hook implementations**: No new hooks added. Existing hooks unaffected. ✓
- **Configuration schema**: `exposed_slots` already has schema defined in `canvas.schema.yml`. New config (field storage, field instance) exported correctly. ✓
- **Entity API usage**: Proper use of `EntityTypeManager`, `ContentTemplate::loadForEntity()`, `EntityTypeBundleInfoInterface`. ✓
- **Security**: Access checks on all new routes. `NodeLayoutAccessCheck` verifies template, field, and update access. `createAccess()` and `listAccess()` check entity-specific permissions. PATCH guard prevents unauthorized template editing. ✓
- **Cache invalidation**: `NodeLayoutAccessCheck::access()` adds node as cacheable dependency. `listAccess()` adds `user.permissions` cache context. ✓

### Code Quality

- **Follows existing patterns**: New code follows the existing Canvas module patterns (FQCN service keys, `_custom_access` usage, `ComponentTreeLoader` delegation). ✓
- **No code duplication**: `getPerContentTemplate()` centralizes per-content mode detection. `getMergedComponentTree()` eliminates merge logic duplication between `build()` and API. ✓
- **Error handling**: Proper exception handling (`LogicException` caught in access checker, `BadRequestHttpException` for invalid requests, `AccessDeniedHttpException` for template component editing). ✓
- **Separation of concerns**: `SlotTreeExtractor` is a focused service for tree extraction. Access check is a dedicated service. ✓
- **unset($component) after foreach-by-reference**: Both locations (lines 108 and 499 in `ApiLayoutController.php`) correctly `unset($component)` to avoid dangling reference issues. ✓

### Integration

- **Existing modules**: Changes work with existing Canvas module ecosystem. `ContentTemplateAwareViewBuilder` rendering unaffected (verified: `build()` now delegates to `getMergedComponentTree()`). ✓
- **Dependencies**: No missing imports. `SlotTreeExtractor` and `NodeLayoutAccessCheck` registered in services.yml. ✓
- **Route conflicts**: No conflicts. `canvas.node.layout` path `/node/{node}/layout` is unique. ✓
- **Config export**: Config exported and appears complete (field storage, field instance, form/view displays, content template). ✓

### Potential Issues

1. **`field_component_tree` in view display** (minor): The field appears in `core.entity_view_display.node.article.default.yml` with `canvas_naive_render_sdc_tree` formatter. The plan specifies it should NOT appear in the default view display. In practice, `ContentTemplateAwareViewBuilder` takes over rendering for nodes with ContentTemplates, so this formatter may be bypassed during normal rendering. However, if the ContentTemplate is disabled or deleted, this formatter would render the raw component tree, which could produce unexpected output. Consider hiding from the view display.

2. **Missing PHPCS fix for new code** (minor): Two new PHPCS errors introduced:
   - `hasContentTemplateWithExposedSlots()` missing `@return` description
   - `updateEntity()` missing `@param` for `$exposed_slots`

3. **Missing direct tests** (minor): `buildPreviewRenderable()` and `buildLayoutAndModel()` in per-content mode are not directly tested as standalone methods. They are exercised indirectly through `testGetMergedModeEditableMetadata` and `testAutoSaveRoundTripSlotData` which test the `get()` method (which calls `buildPreviewRenderable()`). However, `buildLayoutAndModel()` (called by `patch()`) doesn't have equivalent direct testing. The node creation `uid` test in OAuth context is also missing (Plan Step 12 item 12).

4. **`use http\Exception\InvalidArgumentException`** (pre-existing, not introduced by this change): `ApiContentControllers.php` line 38 imports `http\Exception\InvalidArgumentException` from the PECL HTTP extension. This is used in `getUrlFromRoute()` line 497. If the PECL HTTP extension is not installed, this will cause a fatal error. This is pre-existing in the contrib module but worth noting as a risk.
