### Detailed Code Analysis

#### DynamicPropSource.php — evaluate() Logic Flow

The `evaluate()` method at line 144-168 follows this correct flow:

1. **Line 148**: `$raw_result = Evaluator::evaluate(...)` — compute the raw dynamic value
2. **Lines 151-156**: If adapter exists AND value is NOT NULL → adapt and return (preserves existing behavior exactly)
3. **Lines 162-165**: If value is NULL AND fallback exists → use fallback value with raw_result's cacheability (new behavior)
4. **Line 167**: Otherwise → return raw_result (preserves existing behavior)

**Verified edge cases:**
- **Adapter exists but value is NULL:** Falls through adapter block (line 151 `$raw_result->value !== NULL` is false), reaches fallback check. Correct — adapters should not process NULL.
- **Adapter returns non-NULL value:** Returns at line 155, never reaches fallback. Correct.
- **No adapter, no fallback, value is NULL:** Returns raw_result at line 167. This is the pre-existing behavior.
- **Cacheability preservation:** `new EvaluationResult($fallback_result->value, $raw_result)` at line 164 — `$raw_result` implements `CacheableDependencyInterface` (confirmed in EvaluationResult.php). The entity field's cache tags/contexts are preserved even when the field is empty. This is critical for cache invalidation when the field later gets populated.

#### StaticPropSource::evaluate(NULL, ...) Safety

Confirmed at `StaticPropSource.php` line 392-397: the `evaluate()` method evaluates against `$this->fieldItemList` (not the `$host_entity` parameter). Passing `NULL` as host entity is safe.

#### hydrateComponent() Change Impact

Line 391: Changed from `if (!$is_required && $resolved_value->value === NULL)` to `if ($resolved_value->value === NULL)`.

- **$is_required variable still used:** at line 399 for the object-shaped prop special case (`!$is_required && $prop_expression instanceof ObjectPropExpressionInterface`). Not unused after this change.
- **Defense-in-depth rationale:** For required props with DynamicPropSource fallback, `evaluate()` should always return the fallback value (never NULL). This change only protects against edge cases where a required prop resolves to NULL through some other code path.
- **Impact on optional props:** No change — optional props that resolve to NULL were already unset. Now required props that resolve to NULL are also unset, preventing SDC validation errors.
